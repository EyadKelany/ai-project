Core Topics

Generative AI Deep Dives

Technical guides for each major area of generative AI.

LLM Architecture & Training

From tokenization to RLHF — the complete pipeline of modern LLM development.

  • Tokenization: BPE, Unigram, WordPiece — subword vocabularies, 32k–256k tokens
  • Pre-training: Next-token prediction on trillion-token corpora, mixed precision, 3D parallelism
  • Post-training: SFT (instruction tuning), RLHF/DPO/ORPO (preference alignment)
  • Inference: KV cache, speculative decoding, continuous batching, PagedAttention
How LLMs Work Scaling Laws

Prompt Engineering Mastery

Programming in natural language — structured patterns that reliably produce quality outputs.

  • Core patterns: Zero/few-shot, Chain-of-Thought, Self-consistency, Tree-of-Thought
  • Structured output: JSON schemas, function calling, Instructor/Pydantic validation
  • Advanced: DSPy (programmatic optimization), soft prompts, prompt chaining
  • Testing: Prompt regression tests, eval harnesses, A/B testing prompts
Prompt Engineering Guide DSPy

RAG: Retrieval-Augmented Generation

Grounding LLM generation in external knowledge for factual, citeable, up-to-date answers.

  • Pipeline: Chunk → Embed → Index → Retrieve → Re-rank → Generate
  • Chunking: Fixed, semantic, hierarchical, document-aware strategies
  • Advanced: HyDE, query rewriting, multi-hop, GraphRAG, agentic RAG
  • Evaluation: Faithfulness, Answer Relevancy, Context Precision/Recall (RAGAS)
RAG Deep Dive RAGAS

Fine-Tuning & PEFT

Adapting pre-trained models to specific domains, styles, or tasks efficiently.

  • Full fine-tune: All params, massive compute, best for domain adaptation (code, legal, medical)
  • LoRA/QLoRA: Low-rank adapters, <1% trainable params, 4-bit quantization (QLoRA)
  • Tools: Unsloth (2-5x faster), TRL, Axolotl, LLaMA-Factory, PEFT
  • When to use: Style/format consistency, new skills, compression, specialized knowledge
When to Fine-tune Unsloth

AI Agents & Tool Use

Autonomous systems that plan, act, observe, and iterate to achieve goals.

  • Architectures: ReAct, Plan-and-Execute, Reflection, Multi-agent (AutoGen, CrewAI, LangGraph)
  • Tools: Search, code execution, APIs, browsers, databases, file systems
  • Memory: Short-term (context), long-term (vector store), episodic (past runs)
  • Evaluation: SWE-bench, MLE-bench, WebShop, OSWorld, agent-specific benchmarks
AI Agents Guide LangGraph

Diffusion Models & Image Gen

Probabilistic models that learn to denoise — powering Midjourney, DALL-E, Stable Diffusion, Flux.

  • Forward process: Gradually add Gaussian noise until pure noise
  • Reverse process: Neural net predicts noise → denoise step by step
  • Guidance: Classifier-free guidance (CFG), textual inversion, LoRA, ControlNet
  • Modern: Latent diffusion (SD), rectified flow (Flux), DiT (Sora)
How Diffusion Works ComfyUI

Multimodal Models

Models that understand and generate across text, images, audio, video, and 3D.

  • Vision-Language: CLIP, SigLIP, BLIP, LLaVA, GPT-4V, Gemini, Qwen-VL
  • Any-to-Any: ImageBind, CoDi, AnyMAL, Unified-IO
  • Video: Sora, Runway Gen-3, Luma, Kling, Stable Video Diffusion
  • Audio: Whisper, MusicGen, AudioLDM, Suno, ElevenLabs, SpeechT5
Multimodal Guide Models

Evaluation & Safety

Measuring quality, detecting hallucinations, and ensuring responsible deployment.

  • Quality: LLM-as-judge, RAGAS, MT-Bench, AlpacaEval, human preference
  • Safety: Llama Guard, ShieldGemma, constitutional AI, red-teaming
  • Hallucination: SelfCheckGPT, Faithfulness metrics, citation verification
  • Guardrails: Guardrails AI, NeMo Guardrails, structured output validation
Evaluation Methods Guardrails

Local LLMs & Deployment

Running models privately on your hardware — privacy, cost control, offline access.

  • Tools: Ollama, LM Studio, llama.cpp, llamafile, vLLM, TGI, MLX
  • Quantization: Q4_K_M (sweet spot), Q5_K_M, Q8_0, AWQ, GPTQ
  • Hardware: Apple Silicon (unified memory), NVIDIA (CUDA), AMD (ROCm), CPU offload
  • API-compatible: OpenAI-compatible endpoints for drop-in replacement
